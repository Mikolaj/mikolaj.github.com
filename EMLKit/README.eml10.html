The EML Kit

This is an old homepage of the EML Kit. Please update your links to point to http://zls.mimuw.edu.pl/~mikon/ftp/EMLKit/README.html. Thank you!

The EML Kit aims to be (the basis and test-bed for) a comfortable framework for the formal development of programs, using the Extended ML language, formalism and methodology.

Version 1.0 is a complete implementation of the Extended ML seen as a 'programming language'. It allows for parsing, type-checking and evaluation of arbitrary EML code, be it pure Standard ML or full-blown EML with its axioms and other specification constructs. All of this is done in a strict adherence to the formally defined EML Syntax, Static Semantics and Dynamic Semantics.

Now, there is a great need for an "implementation" of the EML Verification Semantics. We hope the EML Kit will help in as well as benefit from the current work on the EML proof theory, proof obligation generators, prototype provers, etc. Yet, as for now only the syntax and type correctness of axioms is checked by the EML Kit - there is no extra aid in proving them correct, sufficient or whatever.

The EML Kit 1.0 Sources Release is intended for everybody (you can fetch the eml10-src.tar.gz archive or look at the README.eml10-src).

The EML Kit 1.0 i386 Binaries Release is for i386 (Linux) users only (you can fetch the eml10-i386.tar.gz archive or look at the README.eml10-i386).

The EML Kit 1.0 SPARC Binaries Release is for Sun SPARC (Solaris) users only (you can fetch the eml10-sparc.tar.gz archive or look at the README.eml10-sparc).

There is also some documentation available. In particular our paper on the EML Kit.

The development of the EML Kit has been partially supported by KBN (Polish State Research Committee) grants " Formal Methods of Software Development" and " Logical Aspects of Software Specification and Development Methods".

The EML Kit is based on the ML Kit; a flexible, clean and modularized interpreter of the Standard ML, written in Standard ML.
